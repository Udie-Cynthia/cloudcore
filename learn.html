Cloud Fundamentals — Overview

Start here to understand what “the cloud” really means, how it’s organized, and how you’ll be billed.

1) What is Cloud Computing?

On-demand access to compute, storage, and networking over the internet — you pay for what you use and skip buying/maintaining hardware.

  • Elasticity: scale up/down quickly.
  • Managed services: the provider runs a lot for you.
  • Global reach: deploy close to users worldwide.

2) Service Models (IaaS, PaaS, SaaS)

Different levels of control vs convenience:

  • IaaS: You manage OS and apps; provider gives VMs, storage, networks (e.g., EC2+EBS).
  • PaaS: You deploy code; provider manages runtime & scaling (e.g., AWS Elastic Beanstalk).
  • SaaS: Use the app; provider manages everything (e.g., managed email, CRM).

3) Deployment Models

  • Public cloud: shared infrastructure, best agility/cost.
  • Private cloud: dedicated to one org (on-prem or hosted).
  • Hybrid: mix of on-prem and public cloud.
  • Multi-cloud: using multiple cloud providers strategically.

4) Global Infrastructure

Clouds are organized into Regions (geographic areas) and Availability Zones (separate data centers in a region).

  • Regions: choose based on latency, compliance, and services.
  • AZs: deploy across 2+ AZs for high availability.

5) Shared Responsibility Model

Security is a partnership:

  • Provider: “of” the cloud — physical data centers, hardware, core services.
  • You: “in” the cloud — identity, access, data protection, OS/app config.

6) Pricing Basics

  • Compute: per hour/second; sized by CPU/RAM (right-size, turn off when idle).
  • Storage: per GB/month; choose classes (e.g., standard vs infrequent access).
  • Data transfer: ingress often free, egress billed (architect to reduce egress).

Tip: Set budgets and alerts early.

7) Core Service Map (Beginner View)

  • Compute: EC2, Lambda
  • Storage: S3, EBS
  • Networking: VPC, Subnets, Internet/NAT Gateway
  • Identity: IAM (users, roles, policies)
  • Databases: RDS (managed relational), DynamoDB (NoSQL)
  • Monitoring: CloudWatch (metrics/logs)

Quick Check (Self-quiz)

  • What’s the difference between a Region and an Availability Zone?
  • Who secures OS patches in IaaS vs PaaS?
  • How can you reduce egress charges in an architecture?
